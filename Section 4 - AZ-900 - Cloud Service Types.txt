Section 4 - AZ-900 - Cloud Service Types

"As a Service"
	- You can rent it for a short time, if you wish
	- No commitment, if you wish
	- Pay for what you use (time,GB, or other)
	- Cloud Provider takes care of buying, developing, and maintaining it.
	
1. Infrastructure as a Service (IaaS)
	- These are the essential services of technology
		- Computing
		- Storage
		- Networking
	
	- Generally have "real world" equivalents in your own data center.
	- Cloud replacements of real world things.
	
	IaaS Computing
		- One example is Azure Virtual Machines
		- Pay by the second
		- Many choices in CPU speeds, RAM, optimazations
	
	IaaS Storage
		- Azure Storage is one example of this
		- 5 PB of storage capacity
		- Can handle, blobs, files, queues, and tables
		- Can also be configured as data lake
		
	IaaS Networking
		- Virtual Networking is an example of this
		- Virtual Networks don't cost anything
		- There are ingress and egress bandwidth costs
			- Ingress - Inbound data into azure (Free)
			- Egress - Data leaving azure (5GB Free monthly)

2. Platform as a Service (PaaS)
	- Cloud service providers have an opportunity to provide more than just the "basic" infrastructure.
	- PaaS includes a service layer on top of IaaS - computing, storage and networking.
	- Middleware, development tools, database server, and more.
	
	PaaS Computing
		- One example is Azure App Services
		- Simply upload your code and configuration to Azure, and it runs your code without needing to worry about the VM underneath.
		- Includes scaling features, CI/CD, containers, staging and development environments, etc.

	PaaS Storage
		- Managed Storage is an example of this.
		- Azure SQL Database is another example.
		- Freed from worrying about the server VM itself or the hard disk itself.

	PaaS Networking
		- Azure Front Door is an example of this.
		- Load Balancer
		- Firewall
		- These are software applications that perform networking tasks.

3. Software as a Service (SaaS)
	- Cloud Apps
	- Tools such as Office 365, OneDrive, Skype
	- The app is ready to be used and you simply need to set it up and use it.

Serverless

What does "serverless" mean?
	- First you don't manage the server.
	- You only handle writing and running the code.

Serverless - Consumption-Based Pricing
	- You only pay for what you use.
	- No charges when your app or database is idle.
	- No need to pre-purchase or reserve resources.

Example - SQL Database Options

Azure SQL Database has multiple pricing options:
	1. DTU-based: Fixed performance levels (S1,S2, etc)
	2. vCore-based: You choose CPU, memory, and storage.
	3. Serverless: Auto-scales CPU & memory based on usage.

Serverless SQL - Key Features
	- Azure auto-scales between 0.5 to 80 vCores.
	- You're charged per vCore-second and storage used.
	- It can pause whne idle, cutting costs entirely.

Example - Azure Functions
Azure Functions is a classic serverless example:
	- First 1 million executions per month are free.
	- After that, you pay per execution and execution time.
	- Azure handles scaling automatically.

Serverless Pros and Cons

Advantages:
	- No server management
	- Scales automatically
	- Cost-efficient for intermittent workloads

Challenges:
	- Costs may vary month-to-month
	- Cold starts may cause slight delays

Serverless means:
	- You focus on code, not infrastructure
	- You pay based on actual usage
	- It's ideal for event-driven or low-usage workloads
